Kubernetes in the clouds¶

Maintaining Kuberentes clusters on-premise can be hazzle. For testing some of the recipes, I highly recommend using a cloud provider as it is easier to follow the cattle not pets approach.

Setting up Kubernetes¶

My tool of choice for setting up volatile infrastructure (servers, clusters, domains, certificates) is Terraform. At GitHub I collected some snippets and links how to set up a basic Kubernetes clusters with different cloud providers.

  • Digital Ocean
  • Google GCP
  • Amazon EKS
  • Azure

My personal favorite (by far) is Digital Ocean as the cluster is up and running in less than 3 minutes time and 20 lines of Terraform code!
